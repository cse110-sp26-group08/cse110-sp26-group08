## Discussion

### Backend Progress and Demo

The backend team gave a progress update and walkthrough of the current implementation. The team was shown how the backend was structured, what had already been connected, and what each major piece was responsible for.

The backend is now connected to MongoDB and supports login flow into the dashboard. A documentation file also exists for the available endpoints, including user-related endpoints that the frontend team may need when retrieving account information for the dashboard.

The current flow demonstrated was:

- Open the app homepage
- Navigate to the login page
- Log into an account
- Reach the dashboard

Some backend pull request work was still pending, so not every connected behavior was fully merged yet.

### Local Setup and Environment Issues

Part of the meeting focused on getting everyone aligned on local setup requirements.

The team discussed that everyone should:

- Pull the latest changes from `main`
- Run `npm install`
- Use the correct backend startup command once setup issues are resolved
- Download the environment file from the Slack environment-files channel

It was noted that some machines were still facing setup problems, including environment variable setup and issues preventing `npm start` from working properly. Those issues were identified as temporary blockers that still needed to be fixed.

### Frontend Progress and Priorities

The frontend team reported that the login page styling had been completed and that login had already been connected to the backend. Based on that progress, the homepage, login page, and sign-in flow were considered mostly complete for the current sprint.

The next priority identified for the frontend team was the dashboard. The main focus should now be:

- Connecting dashboard functionality to the backend
- Improving dashboard styling and presentation

The wireframe work was also reported as complete at a basic level, though follow-up comments were raised about missing details.

### Dashboard and App Flow Feedback

Several UI and flow concerns were raised during the discussion.

Current issues identified included:

- The homepage includes duplicate calls to action for login or signup
- The dashboard should include a logout option
- After login, the app currently opens directly into one dashboard view
- If a user has multiple apps, the product should not default immediately into one app dashboard without a selection step

A suggested improvement was to add an intermediate apps page. This page would behave similarly to a dashboard selector, where users first see a list of their available apps and then choose which one to open. After that selection, the app-specific dashboard would load.

### Wireframe Expectations

The meeting also revisited wireframe quality and planning expectations.

Feedback from the professor was shared: planning documents and wireframes should define the website ahead of implementation rather than simply being updated to match code after the fact. The wireframe should serve as the blueprint for what gets built.

Issues noted with the current wireframe included:

- Some sections still use placeholders
- A signup page is missing
- More complete planning should exist before further implementation

The team was advised to make sure the wireframe fully represents the intended product flow and page set before using it as the basis for continued frontend work.

### Collector and Tester App Updates

The collector script was reported as nearly complete. It already handles error tracking and performance tracking, with only a few minor fixes still needed.

The tester app was also reported as complete in its current form. With help from the backend team, the tester app and collector were successfully connected to the backend server locally. Some smaller bugs and functional improvements still remain, but the core connection flow is working.

It was also mentioned that the tester app may be pushed later from a separate repository into the team workflow.

### Code Quality and PR Workflow Notes

The discussion included a small code quality suggestion for the collector script. One function definition pattern in the collector file was described as unnecessarily indirect. Instead of assigning a collector function to a variable and then calling it, the recommendation was to simplify the file by directly running the function.

The team also discussed pull request communication habits. Rather than talking extensively about PRs only in Slack, the group should move more of that discussion into the pull request threads themselves. This helps preserve review history and makes the reasoning behind requested changes visible to TAs and graders.

Benefits of using PR threads more directly include:

- Clearer review context
- Easier visibility into why changes were requested
- Better documentation of revision history
- Less confusion when reviewing many commits in one PR

## Takeaways and Summary

This standup clarified both technical progress and several workflow improvements the team still needs to make.

1. **Backend integration is advancing:** login flow, MongoDB connection, and endpoint documentation are in place
2. **Local setup still needs cleanup:** some teammates still need environment configuration and startup issues resolved
3. **Frontend login work is mostly complete:** dashboard integration and styling are now the main priorities
4. **App navigation needs improvement:** the team should likely add an app-selection page before loading a dashboard
5. **Wireframes need to lead implementation:** missing pages and placeholders should be addressed before continued UI development
6. **Collector and tester app work are nearly done:** both are functional, with only minor fixes remaining
7. **PR discussions should move into GitHub:** review comments and requested changes should live in PR threads rather than only in Slack

**Key follow-up items:**

- Pull the latest `main` branch changes on all machines
- Download and configure the shared environment file from Slack
- Fix the backend startup issues affecting local setup
- Continue dashboard styling and backend connection work
- Expand the wireframe to include all required pages and flows
- Clean up the collector implementation
- Keep PR review discussion inside GitHub pull requests
